Issue Report: CPU Thermal Throttling on High Workload

Issue Summary

HSD No.: 2765984002

Title: CPU Thermal Throttling During Sustained High Workload

Description: When running a sustained high workload, the CPU experiences thermal throttling, leading to performance degradation. This results in slower computation speeds and potential system instability.

Hardware Configuration

  • Hardware Model: Lenovo ThinkSystem SR650
  • CPU: Intel Xeon Gold 6130 (2x)
  • Cooling System: Standard Air Cooling
  • Memory: 256 GB RAM
  • Network Adapter: Broadcom NetXtreme 10G
  • Firmware Version: 1.9.2

Software Configuration

  • Operating System: Ubuntu 20.04 LTS
  • Kernel Version: 5.4.0-90-generic
  • Thermal Management Tool: lm-sensors 3.6.0
  • Network Driver Version: bnxt_en 1.10.0
  • Workload Management: stress-ng 0.11.24

Steps to Reproduce

  1. Install Ubuntu 20.04 on the Lenovo ThinkSystem SR650 server.
  2. Configure CPU performance scaling using cpufrequtils.
  3. Run a stress test using stress-ng --cpu 48 --timeout 600 to generate sustained CPU load.
  4. Monitor CPU temperature and frequency using lm-sensors and watch -n 1 sensors.
  5. Observe the CPU frequency reduction and increased temperature.

Expected Result

The CPU should maintain stable performance during the sustained workload without significant thermal throttling or performance degradation.

Actual Result

The CPU begins throttling after a few minutes of stress, with frequency dropping significantly and performance degrading due to high temperatures.

Attached Files

  • Thermal Readings
  • CPU Frequency Statistics
